CSS preprocessing can be implemented in two ways. Client-side preprocessing compiles the preprocessor file (like a .scss file) directly in the browser at runtime, which requires linking a JavaScript compiler in your HTML. While convenient for quick prototyping, it adds load overhead and is not recommended for production. Server-side preprocessing (the more common and preferred approach) compiles your .scss files into plain .css files ahead of time — before they're ever served to a browser. Tools like Dart Sass, Node.js scripts, or GUI apps like CodeKit handle this compilation step, producing optimized CSS that loads faster and requires no client-side processing.


Sass is a professional-grade CSS extension language sass-lang that introduces several powerful features:


Variables

Store reusable values like colors, fonts, or spacing. Instead of hardcoding #3a86ff across dozens of rules, you define $primary-color: #3a86ff once and reference it everywhere — making global updates instant.


Mixins

Reusable blocks of CSS declarations you define once and include anywhere with @include. They're especially useful for vendor prefixes or repeated style patterns like button styles or flexbox layouts.


Functions

Let you compute and return values — for example, calculating a percentage-based width or manipulating a color's lightness programmatically. Sass includes built-in functions (like darken() and lighten()) and lets you define custom ones with @function.


Operators

Allow math directly in your stylesheets — addition, subtraction, multiplication, and division — so you can write things like width: 100% / 3 instead of manually calculating 33.33%.


Nesting

Lets you write CSS that mirrors the structure of your HTML, placing child selectors inside their parent selectors. This reduces repetition and makes stylesheets far more readable. For example, writing nav { ul { list-style: none; } } compiles cleanly to nav ul { list-style: none; }.
